# ACCEPTANCE CRITERIA

Объект принимается только если выполняются все условия.

## A. Геометрия
- соблюдены основные пропорции по чертежам
- читается вид сверху
- читается фронтальный профиль
- читается боковой профиль
- нет лишней псевдореалистичной геометрии
- нет бессмысленной детализации

## B. Шаблонность
- выбран template family
- recipe не противоречит template
- объект разбит на логичные части

## C. Редактируемость
- есть source recipe
- есть part graph
- есть material slots
- есть sockets
- есть pivots
- есть deform zones
- есть animation hooks

## D. Продакшен пригодность
- poly budget соблюдён
- naming последовательный
- pivot корректный
- scale корректный
- runtime builder работает
- объект ставится в сцену без ручной правки

## E. Визуальная читаемость
- узнаётся в изометрии
- узнаётся в ортографии
- силуэт чистый
- крупные массы не потеряны
- деталь не шумит

## F. Что обязательно приложить к результату
1. build-файл
2. recipe-json
3. краткое описание принятых интерпретаций
4. список hooks
5. tri count
